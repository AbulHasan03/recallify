📐 Scoring Formula
match_score(u1, u2) =
  0.40 × cosine_sim(u1.embedding, u2.embedding)
+ 0.25 × goal_match(u1.learning_goal, u2.learning_goal)
+ 0.15 × cadence_sim(u1.activity_pattern, u2.activity_pattern)
+ 0.12 × skill_proximity(u1, u2)
+ 0.08 × schedule_overlap(u1.timezone, u2.timezone)

🔄 Embedding Pipeline
1
User watches/uploads video → AI extracts topic tags + concepts
2
Tags mapped onto Interest hierarchy; affinity scored by dwell time
3
User embedding = weighted average of interest embeddings (pgvector)
4
ANN search (HNSW index) finds top-50 candidates in <10ms
5
Full scoring formula re-ranks candidates; top-5 surfaced in UI
🛡️ Privacy Guardrails
✓
Anonymous mode: embedding computed but identity never revealed to matches
✓
Friends-only: only users with mutual follows enter the candidate pool
✓
Opt-out flips a flag: embedding removed from ANN index within 5 minutes
✓
No raw watch history ever shared — only derived embedding vectors
🏗️ System Overview
1
WebSocket Gateway — connection manager (Socket.io / Ably). Each user holds one WS connection, multiplexed across rooms.
2
Room Service — manages room lifecycle (create, join, leave, archive). Rooms are scoped to topic, group, or video.
3
Message Bus — Redis Pub/Sub fans out messages to all subscribers. Guarantees delivery under 100ms p99.
4
Persistence — messages written async to Postgres. Timestamps linked to video position for inline Q&A.
5
AI Moderator — runs on message stream; auto-generates discussion prompts, flags off-topic drift.
📡 Watch Together Mode
1
Host creates Watch Room → server assigns a playback session ID
2
All members join via WebSocket; host controls play/pause/seek
3
Sync events broadcast via Room Service with <50ms latency
4
Each member's client applies NTP-style clock correction to stay in sync
5
Timestamp-anchored chat: messages tagged to video position, shown in sidebar
